过去一段时间，我一直在写教程长文。

内容越写越完整，步骤越拆越细，但我始终觉得缺了点什么。

文字很多，信息也足够，可读者打开文章以后，依然需要花很长时间理解。一整屏文字连续往下铺，哪怕内容本身有价值，也很容易产生阅读压力。

尤其是教程、工作流和方法论文章，很多内容其实并不适合只靠文字表达。

一个流程如何运转，一个工具怎样处理信息，一个人在某种处境里遇到了什么问题，这些东西本来是可以直接“看见”的。

所以我开始测试留下点视觉印记——长文插图。

我测试了很多种人物风格、场景结构、画面比例和信息密度，最后才逐渐确定了现在这套方案：

## 先固定个人IP，再生成IP小剧场

## 第一步，用自己的照片生成一张稳定的IP角色设定图

![图像](https://pbs.twimg.com/media/HNKWIk7bsAA4CMh?format=jpg&name=large)

```text
请根据我上传的人物照片，生成一张“复古扁平立体角色设定图”。

角色名称：{{填写名字或昵称}}

把照片中的真人转化为可长期复用的个人IP角色，做成一张竖版角色设定板。整体采用复古扁平立体插画风：扁平为主，轻微立体，低细节，低饱和，柔和阴影，哑光质感，轻微复古颗粒，浅米白或暖白背景，版式干净，留白充足，像专业角色设计手册页。

人物必须以我上传的照片为唯一参考，保留最有辨识度的特征，包括脸型、五官比例、眼睛气质、眉毛、鼻子、嘴唇、发型、发际线、年龄感、肤色、体型轮廓和整体气质。表现方式是扁平化角色转译，不要写实，不要二次元，不要潮玩，不要过度可爱，不要大头娃娃。

服装参考原照片，可适度简化成更适合长期使用的固定角色造型。要求简洁、日常、有辨识度，减少复杂褶皱和真实布料纹理，不加无关品牌标志和多余配饰。

固定版式：
1. 顶部居中写“{{填写名字或昵称}}”
2. 标题下方小字写“角色设定”
3. 中上部放一个最大的主角色全身像，站在简洁圆形底座上，姿态自然放松
4. 底部横向排列四个小角色视图，依次为：
正面 / FRONT
左侧 / LEFT SIDE
背面 / BACK
右侧 / RIGHT SIDE

四视图必须保持完全一致：同一个发型、脸型、衣服、鞋子、配件、颜色和身体比例。

不要左侧配件区，不要配色区，不要人物介绍，不要性格说明，不要长段文字，不要多余英文，不要乱码，不要复杂场景。

最终输出为一张清晰、规整、可长期复用的个人IP角色设定图，比例为3:4或4:5。
```

# 第二步：生成IP小剧场

使用小剧场提示词时，需要同时提供两项内容：

```text
1. 上传已经确认的个人IP角色设定图
2. 输入一个主题、观点或一段文字
```

角色设定图是人物身份标准。生成新场景时，必须保持人物的脸型、发型、服装、配色、鞋子和配件不变，只改变动作和姿态。

## 一鱼两吃提示词：

- 生成流程拆解图
- 生成核心动作

```text
请根据我上传的个人IP角色设定图，以及下面提供的主题或文字，生成一张“复古彩色扁平3D IP小剧场”长文插图。

主题或文字：
{{粘贴主题、观点或段落}}

模式：
{{流程拆解 / 核心动作 / 自动判断}}

画幅固定为横版 16:9。背景必须是纯白色页面，不使用米黄色、渐变、纹理背景或完整空间背景。画面用于长文正文插图，不做封面，不放大标题。主体小剧场控制在画布中间区域，整体占画面约 55%—65%，不能过大，不能贴边，四周必须保留明显留白。整体像摆放在白色页面上的一组精致桌面模型，清爽、克制、有呼吸感。

必须严格按照上传的IP角色设定图还原人物，完整保留脸型、五官比例、发型、发际线、肤色、固定服装、鞋子、配件、整体气质和身体比例。不得重新设计人物，不得更换服装，不得增加设定图中没有的显眼配件。允许变化的只有动作、姿态、方向、视线和轻微表情。无论人物在画面中出现几次，都必须保持完全一致。

先理解输入内容，再提炼最重要的结论。不要逐句翻译原文，也不要把所有信息都塞进画面。如果内容本身是一个过程、步骤、方法、工作流或输入到输出的完整链路，则使用“流程拆解模式”；如果内容本身是一个观点、判断、冲突、处境或现实问题，则使用“核心动作模式”。如果用户选择自动判断，就先判断哪种模式更合适，再生成对应画面。

在流程拆解模式下，把内容压缩成 3—5 个关键节点。每个节点只表现一个动作和一个结果，使用统一风格的复古设备、家具、文件或其他真实物件表达。固定IP人物要参与关键节点，不能只是站在一旁。节点之间可以有简洁但克制的连接关系，阅读顺序清楚，但不要做成拥挤的PPT流程图。节点之间必须保留足够白色间距，整体保持节奏感和呼吸感。

在核心动作模式下，把整段内容压缩为一个最值得被看见的物理现场。画面只保留一个固定IP人物、一个核心物件或紧凑物件组、一个明确动作和一个可见结果。通过人物和物件之间的物理关系表达核心观点，不展开完整过程，不堆砌信息，不做复杂解释。整体必须更简洁、更集中、更有留白，坚持 less is more。

整体视觉风格统一采用复古彩色扁平3D编辑插画风格。人物与物件以扁平块面为主，带轻微立体体积和轻等距视角。结构简洁，低细节，边缘清楚，使用柔和阴影、哑光材质和少量复古颗粒。颜色可以比基础版稍微更丰富一点，但仍然必须统一、克制、清爽，不花、不杂、不乱。推荐从橄榄绿、芥末黄、复古蓝、珊瑚红、暖灰、奶油白、鼠尾草绿、浅卡其等颜色中选择 3—5 个主色进行搭配，让画面更有层次，但不能变成高饱和海报感。

画面中的文字分为两类。第一类是物件标签，用来辅助理解物件功能。核心动作模式最多保留 3 个标签；流程拆解模式每个节点最多 1 个标签。标签控制在 2—6 个汉字，并尽量写在文件、机器、门牌、按钮、出口牌、便签或卡片上，而不是漂浮在空中。

第二类是画面内部的简短说明文字，这不是放在画面下方的说明区，而是直接整合在画面里，像 plog 一样，以纸条、便签、贴纸、小卡片、胶带标签或简洁注释卡的形式出现。它们应当自然地分布在主体旁边或留白区域，像是在白色页面上补充说明。根据内容保留 1—2 条必要说明即可，每条控制在 8—20 个汉字，作用是点明结论、补足语境、提高理解效率。说明文字必须短、准、轻，不要写成长段话，不要变成大标题海报，也不要把画面塞满文字。

最终画面必须满足：固定IP没有变化；画幅比例固定为横版 16:9；背景是纯白页面；主体比例适中；四周留白明显；颜色比原来稍微丰富一点但整体仍然克制统一；必要的文字说明直接整合在画面内部；核心内容可以在 3 秒内看懂；没有无关装饰；没有把画面做成复杂信息图；没有为了说明完整而堆积过多元素。
```
