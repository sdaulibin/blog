AI 时代，人人都需要一个自己的知识库。

为什么？

因为 AI 本身没有立场、没有经历、没有审美。

直接问它一个问题，回答正确但没有灵魂；

你想让它替你写一篇文章、做一份方案、整理一段录音——它给你的永远是通用、平均、谁都能写出来的内容。

差别在哪？

**在于你能不能把"你自己的素材、方法论、经历、判断"喂给它。**

同样的模型，喂进去你的东西，输出的就是带你印记的东西。

这就是为什么从今往后，每个人——不只是程序员、不只是知识工作者——都该拥有一座属于自己的本地知识库。

而本地知识库的最佳方案，就是 **Obsidian，全景图：**

![图像](https://pbs.twimg.com/media/HI0cAvlaQAEFeVs?format=jpg&name=large)

每一层都是知识库的一块拼图，缺了任何一块都会有短板。下面按这条路线一层一层来。

## 第 1 章 Obsidian 是什么

**一句话：本地的、基于 Markdown 文件的笔记软件，最大特色是双向链接和插件生态。**

和普通笔记软件有三个本质区别：

1. **本地优先。** 所有笔记是你电脑上的 .md 文本文件，不依赖任何云端。Obsidian 公司倒闭你的笔记照样能用记事本打开。
2. **双向链接。** A 链接到 B，B 也知道 A 引用了它。笔记不再是孤立的文档，而是会织成网。
3. **插件无限扩展。** 上千个社区插件，从日历看板到 AI 接入，想要什么基本都有。

**收费吗？** 个人使用完全免费。只有「云同步」和「在线发布」两个付费服务，新手用不到。

**适合谁？** 笔记体量大、希望长期沉淀知识的人。如果你只是偶尔记购物清单，备忘录就够。

## 第 2 章 下载安装

到 [obsidian.md/download](https://obsidian.md/download) 下载，按系统点对应按钮。

**Windows：** 下载 .exe → 双击安装 → 一路下一步 → 桌面会出现紫色图标。

**macOS：** 下载 .dmg → 双击打开 → 把 Obsidian 拖到 Applications 文件夹 → 启动台打开。

**iOS / Android：** App Store / Play Store 搜「Obsidian」安装。建议先在电脑上把基本用法跑通，再装手机版。

创建第一个仓库

启动后不是空白页，而是让你**创建仓库**或**打开仓库**。

> **仓库（Vault）就是一个普通文件夹。** 你以后写的所有笔记都保存在里面。

操作：选「创建新仓库」→ 取名字（随便，比如"我的笔记"）→ 选位置（桌面或文档文件夹，别放 C 盘深处）→ 创建。

把界面改成中文

设置（左下齿轮）→ **About** → **Language** 选「简体中文」→ **Relaunch** 重启。

## 第 3 章 界面与四个核心概念

界面五大区

- **最左侧竖条：** 工具栏，一列功能图标（文件、搜索、书签、关系图谱）。
- **左侧面板：** 文件管理区，仓库里所有笔记和文件夹。
- **中间区域：** 编辑区，写笔记的地方，支持多标签页。
- **右侧面板：** 反向链接、大纲、标签等辅助信息。第一天可以不管。
- **底部状态栏：** 字数、光标位置。忽略。

概念一：仓库（Vault）

一个文件夹，所有笔记存里面。新手**先用一个仓库**就够，别一上来建好几个。

概念二：Markdown

一种用纯文字标记格式的写法。在 Word 里加标题要点工具栏；在 Markdown 里只需要在文字前打 # 加空格。语法很少，用着用着自然就会，第 4 章给你一份速查。

概念三：双向链接（核心）

在 A 笔记里写 \[\[B笔记\]\]，两件事发生：

1. B笔记变成可点击的链接，点一下跳过去。
2. 打开 B 时，右侧面板会出现一条**反向链接**，告诉你 A 引用了它。

这就是"双向"。它让笔记之间的关联**显性化**——笔记越多，这张网越值钱。

**新手记一件事：在笔记里用 \`\[\[词\]\]\` 把一个词括起来，就创建了一个链接。** 如果这篇笔记不存在，点击它会自动新建。

概念四：插件

- **核心插件：** 官方内置，去设置里开关。
- **社区插件：** 第三方做的，从插件市场装。

**第一天不要碰社区插件。** 先把核心插件里基础的几个用起来，等用了一两周发现某个原生满足不了的需求再去搜。

## 第 4 章 写第一篇笔记

创建笔记

快捷键 Ctrl/Cmd + N，或者左侧文件列表右键 → 新建笔记。

标题随便取，「今天的想法」「读书笔记」都行。**先写起来再说，不要纠结。**

Markdown 速查

```text
# 一级标题
## 二级标题
### 三级标题

**加粗**       *斜体*

- 无序列表项
1. 有序列表项

- [ ] 待办未完成
- [x] 待办已完成

> 引用文字

\`行内代码\`

​
```

python def hello(): print("代码块") ​

```text
[外部链接](https://example.com)
[[内部双链]]
[[内部双链|显示成这段]]

---  （分割线）
```

加粗 / 斜体也可以用 Ctrl/Cmd + B 和 Ctrl/Cmd + I。

三种编辑模式

- **实时预览（默认推荐）：** 打字时符号自动渲染成最终样式。
- **源码模式：** 能看到所有原始符号。
- **阅读模式：** 完全渲染，无法编辑。按 Ctrl/Cmd + E 切换实时预览 / 阅读。

标签和属性

- **标签**：在正文写 [#读书笔记](https://x.com/search?q=%23%E8%AF%BB%E4%B9%A6%E7%AC%94%E8%AE%B0&src=hashtag_click) [#习惯](https://x.com/search?q=%23%E4%B9%A0%E6%83%AF&src=hashtag_click)，点击标签能聚合相关笔记。
- **属性（YAML Frontmatter）**：写在笔记最顶部，用三个减号包裹，记录结构化元数据。新手暂不用学，等做分类汇总时再回头看。

```yaml
---
tags: [读书]
作者: 某某
评分: 5
---
```

## 第 5 章 必学的高频功能

文件、搜索、书签

- **文件列表（左侧）：** 右键新建笔记/文件夹，拖拽移动，双击打开。**别花两小时设计文件夹，先写起来。**
- **搜索（放大镜）：** 在所有笔记中搜关键词，是你以后用得最多的功能之一。
- **书签（星标）：** 每天要看的笔记加书签，避免去文件列表里翻。

两个必记肌肉记忆

- **\`Ctrl/Cmd + P\` 命令面板：** 万能搜索器。任何功能找不到就在这里搜（创建、设置、导出、运行插件……都能搜到）。
- **\`Ctrl/Cmd + O\` 快速切换：** 输入笔记名秒开。比鼠标翻文件夹快十倍。

关系图谱

可视化所有笔记的连接关系。**笔记少时基本没用**（几个零星点），等写到几百篇会非常震撼。新手看一眼就行，别花时间研究。

必调的几个设置

- 编辑器 → 默认编辑模式 → **实时预览**。
- 文件与链接 → 新建笔记存放位置 → **指定一个收集箱文件夹**，否则到处乱跑。
- 文件与链接 → 附件默认存放路径 → **指定文件夹**，比如 附件，所有粘贴/拖入的图片 PDF 统一进这个文件夹。
- 核心插件里确认开启：日记、大纲、书签、标签、模板。

高频快捷键

![图像](https://pbs.twimg.com/media/HI0YsAWa8AAXT41?format=jpg&name=large)

更多在「设置 → 快捷键」里查看和自定义。

## 第 6 章 笔记组织的三套方法论

**核心原则：** 文件夹保持浅层（不超过 1-2 级），靠链接和标签做主组织，不要花几小时设计完美分类——**没写过几篇笔记就开始设计分类，必翻车。**

MOC（内容地图）

建一篇"目录笔记"，里面主要不是写正文，而是把同主题的其他笔记按结构列出来：

```text
# Python 学习地图
## 基础
- [[变量与数据类型]]
- [[函数]]
## Web 框架
- [[FastAPI 入门]]
```

适合做主题入口。还可以建一篇 Home 笔记，列出所有 MOC，形成"地图的地图"。**最受欢迎的 Obsidian 组织方式。**

PARA

把所有笔记按**用途状态**分四类：

- **Projects 项目：** 正在推进、有终点的事（"装修房子"）。
- **Areas 领域：** 长期负责、没有终点（"健康""理财"）。
- **Resources 资源：** 兴趣素材库（"摄影""Python"）。
- **Archives 归档：** 已结束、不活跃。

迁移技巧：从其他工具搬过来时，**先把旧笔记全扔进 Archives**，你就有了一张干净的桌面，但历史都还在。

Zettelkasten（卡片盒）

核心三条：每篇笔记**只写一个想法**、**用你自己的话**重新表达、**主动建立链接**。流程是「随手记 → 定期整理 → 重写并链接」。适合经常写作、做研究的人。**它不是分类法，是写作法。**

怎么选

- 想直观看到主题结构 → MOC
- 想按"正在做什么"组织 → PARA
- 想沉淀写作素材 → Zettelkasten

**最好的方法是先写半个月，看哪种需求自然冒出来，再有意识地用对应方法。**

## 第 7 章 必装的 10 个社区插件

7.1 通用安装方法

设置 → 社区插件 → 「关闭受限模式」→ 「浏览」打开插件市场 → 搜索 → 「安装」→ **点「启用」**（很多人忘了点启用以为插件坏了）。

**如果插件市场打不开：** 切换网络、用加速工具，或去 PKMer 等国内镜像站手动下载插件文件，放到 仓库路径/.obsidian/plugins/插件名/ 目录下重启 Obsidian。

下面 10 个插件按推荐安装顺序排列。其中 **Templater、Tasks、QuickAdd、Dataview** 配置稍复杂，详细展开；其余 6 个装上就能用，只给一句话简介。

7.2 第一批（立刻装）

**Calendar（日历）** —— 右侧栏多一个可视化月历，能直观看到哪天写了日记、哪天没写。点日期跳转到/创建那天的日记。必须配合核心插件「日记」。把日历面板钉在右侧栏，每天打开 Obsidian 第一件事就是点今天 → 写日记。

Templater（高级模板）——重点

比核心模板插件强大得多。能在模板里自动填入当前日期、笔记标题、星期几，甚至弹出选择框让你选。

**怎么用：**

1. 设置 → Templater → Template folder location，填入存放模板的文件夹名，比如 模板。
2. 在「模板」文件夹里新建一篇笔记当模板，比如「日记模板」，内容：

```text
# <% tp.date.now("YYYY-MM-DD") %> 日记

> 今天是 <% tp.date.now("dddd") %>。

## 今天发生了什么

## 今天学到了什么

## 明天要做什么

- [ ]
```

1. 新建笔记后按 Ctrl/Cmd + P → 搜「Templater: Insert template」→ 选模板 → 模板内容自动填入，日期会被替换成今天。

**常用变量：**

![图像](https://pbs.twimg.com/media/HI0YsjUawAA_MEw?format=jpg&name=large)

**实操建议：** 给最常用的几种笔记各做一个模板：日记、读书笔记、会议记录、项目笔记。

**Image Toolkit（图片工具箱）** —— 点击笔记里的图片弹出全屏预览，支持鼠标滚轮缩放、拖拽移动、左右键切换、Esc 关闭。**装完即用，不需要配置。** 笔记里经常贴截图的人必装，看图体验从"原始"变成"正常"。

7.3 第二批（用了一周后装）

Tasks（任务管理）——重点

Obsidian 原生支持 - \[ \] 任务列表，但散落在不同笔记里没法统一管理。Tasks 让你能在任何一篇笔记里写查询，自动汇总所有未完成任务。

**任务写法：**

```text
- [ ] 写周报 📅 2026-05-22
- [ ] 整理项目文档 ⏳ 2026-05-25
- [x] 已完成的任务
```

📅 是截止日期，⏳ 是计划日期。不用手敲 emoji——光标放在任务行 → Ctrl/Cmd + P → 「Tasks: Create or edit task」→ 弹出可视化编辑器。

**任务汇总视图：** 在任意笔记里写：

\~~~

```text
not done
due before tomorrow
```

\~~~

保存后会自动列出所有未完成的、明天前到期的任务。

**实操建议：** 建一篇「任务总览」笔记，放几个不同条件的查询块（今天到期、本周到期、所有未完成）。每天打开它，全局任务一目了然。

QuickAdd（快速添加）——重点

把"灵感闪过 → 想记下来"的摩擦力降到最低。按一个快捷键 → 弹输入框 → 打字回车 → 内容自动追加到指定笔记。整个过程不超过 3 秒。

**典型配置（快速捕捉灵感）：**

1. 先在仓库里建一篇「灵感收集箱」笔记。
2. 设置 → QuickAdd → 在顶部输入"捕捉灵感" → 类型选 Capture → 点 Add Choice。
3. 点新建项右边的齿轮 → 配置：
- File Name：灵感收集箱
- Capture format：- {{VALUE}} ({{DATE:YYYY-MM-DD HH:mm}})
1. 回到 QuickAdd 设置首页 → 点这一项右边的闪电图标 ⚡ 注册为命令。
2. 设置 → 快捷键 → 搜「QuickAdd 捕捉灵感」→ 绑一个快捷键，比如 Ctrl/Cmd + Shift + I。

以后任何时候有想法：按快捷键 → 输入内容 → 回车，灵感就乖乖躺在「灵感收集箱」里等你以后整理。

**Periodic Notes（周期笔记）** —— 核心只有日记，这个插件补齐周记、月记、季记、年记。设置里开启 Weekly Notes，Format 设为 YYYY-\[W\]ww，指定模板和文件夹即可。配合 Calendar 可以在日历面板直接点周数创建周记。每周日花 15-20 分钟写一篇周回顾，坚持一个月你会惊讶自己原来做了这么多事。

7.4 第三批（用了两周后装）

**Kanban（看板）** —— 在 Obsidian 里直接创建 Trello 风格的看板：命令面板搜「Kanban: Create new board」创建，添加列（例如「灵感池 → 写作中 → 已完成 → 已发布」）、添加卡片、拖拽。**进阶用法：卡片内容写 \`\[\[笔记标题\]\]\` 就能跳转到对应笔记**，看板和笔记关联起来。

Dataview（数据视图）——重点

Obsidian 最强大的插件之一。把你的笔记当数据库查询。

简单用法举例。先给读书笔记加 frontmatter：

```yaml
---
tags: 读书笔记
书名: 思考快与慢
作者: 某某某
评分: 5
读完日期: 2026-05-15
---
```

然后在任意笔记里写：

\~~~

```text
TABLE 作者, 评分, 读完日期
FROM #读书笔记
SORT 读完日期 DESC
```

\~~~

保存后会自动生成一张包含所有读书笔记的表格，按读完日期排序。**以后每新增一篇读书笔记，表格会自动更新。**

**其他常用查询：**

```text
列出最近 7 天创建的笔记：
LIST FROM "" WHERE file.cday >= date(today) - dur(7 days)

列出所有引用了某篇笔记的笔记：
LIST FROM [[某篇笔记]]
```

**学习建议：** 不要一上来啃所有语法。从 TABLE 和 LIST 起步，遇到具体需求再查文档。可以建几篇「索引笔记」（读书索引、项目索引），里面放对应的 Dataview 查询，分类目录自动生成。

**Excalidraw（白板绘图）** —— 手绘风格的白板，画思维导图、流程图、概念图都很顺手，拖拖拽拽就能画。命令面板搜「Excalidraw: Create new drawing」创建画布。**图形里的文字可以包含 \`\[\[笔记标题\]\]\` 双链**，按 Ctrl/Cmd 点击就跳转。适合梳理复杂主题、做读书笔记概念图。

**Various Complements（智能补全）** —— 打字时根据仓库现有笔记标题弹出建议，按 Tab 补全。最实用的场景：想加 \[\[\]\] 链接但记不清完整标题。**笔记数量上百以后用着才香。** 建议设置：Match strategy 选 prefix、Min word length 设 2-3，避免每打一个字就弹窗。

7.5 安装节奏建议

- **第一周：** Calendar + Templater + Image Toolkit。覆盖"看日历、用模板、看图片"三个基础需求。
- **第二周：** Tasks + QuickAdd + Periodic Notes。当你有任务管理需求、想快速捕捉碎片、想写周回顾时再装。
- **第三周起：** Kanban + Dataview + Excalidraw + Various Complements。这四个功能更强但学习成本也更高，等基础用熟了再装。

**插件不是装得越多越好。** 很多人装了二三十个插件，每个都浅浅试一下，结果 Obsidian 启动变慢、界面变复杂、大部分插件根本没用上。

正确思路：**先遇到问题，再找插件解决。** 而不是"看着厉害先装上"——装了不用等于没装。工具是为你服务的，不是让你为工具打工的。**先把笔记写起来，所有插件都是为"写"服务的。**

## 第 8 章 10 个实操场景模板

**别一次全用上**，挑 1-2 个最有用的先做起来。下面只给模板，操作步骤前面已经教过。

1\. 每日日记（核心插件「日记」启用即可）

```text
## 今天做了什么
-
## 今天学到了什么
-
## 明天要做
- [ ]
```

2\. 灵感收集箱

## 第 9 章 导入外部资料

Word

最省事的方式：装 Importer 插件 → 命令面板搜「Importer: Import data」→ 选 Microsoft Word → 选文件 → Import。

少量文档也可以直接打开 Word Ctrl + A 全选复制 → Obsidian 新建笔记粘贴，标题、加粗、列表会自动转 Markdown。

飞书文档

打开文档 → 右上角「⋯」→ 导出 → 优先选 **Markdown**（没有就选 Word）。Markdown 文件直接拖进仓库即可。

批量导出：飞书云文档勾选多个文件 → 导出为 Word → 解压 → 用 Importer 批量导入。

PDF

**设置 → 文件与链接 → 附件默认存放路径** 指定一个文件夹（如 附件）。然后直接把 PDF 拖进 Obsidian 编辑区，自动入库 + 生成链接。在文件列表双击 PDF 可直接阅读翻页。

笔记里嵌入：输入 !\[\[文件名.pdf\]\]，阅读模式下会嵌入显示。

音频与视频

支持的格式：mp3 / wav / m4a（音频）、mp4 / webm（视频）。其他格式先用 [cloudconvert.com](https://cloudconvert.com/) 这类工具转格式。

放到仓库里，笔记写 !\[\[录音.mp3\]\] 或 !\[\[演示.mp4\]\]，阅读模式会出现播放器。

> **重要：** 视频通常很大，会让仓库膨胀、同步变慢。**只放小视频，大视频用网盘存、笔记里记链接。**

## 第 10 章 接入 AI 大模型

让 AI 直接在 Obsidian 里帮你总结、翻译、改写、写大纲、回答关于你笔记的问题。

几个基础概念

- **API Key：** AI 厂商给你的"钥匙"，平台靠它识别你、扣费。
- **Base URL：** AI 服务的"地址"。
- **Model Name：** 同一家有多个模型，得指定哪一个。

安装 Copilot 插件（所有方案通用）

设置 → 社区插件 → 搜 Copilot（作者 Logan Yang）→ 安装 → 启用。右侧栏会出现聊天气泡图标，点开就是 AI 面板（先配 Key 才能用）。

推荐方案：DeepSeek

国内最强模型之一，价格极便宜（比 OpenAI 便宜几十倍），注册送免费额度，**不需要科学上网**。

**获取 Key：**

1. 浏览器打开 [https://platform.deepseek.com/](https://platform.deepseek.com/) → 注册登录。
2. 控制台左侧 → 「API Keys」→「创建 API Key」→ 取名 → 创建。
3. **\`sk-\` 开头的字符串立刻复制保存**——只显示一次。

**在 Copilot 配置：** 设置 → Copilot → 「Add Custom Model」→ 填：

![图像](https://pbs.twimg.com/media/HI0YtI1bkAAZODG?format=jpg&name=large)

保存 → 把「Default Model」切换为 DeepSeek。

**验证：** 右侧聊天面板输入「你好，自我介绍一下」→ 几秒后回复 = 成功。

其他备选方案

![图像](https://pbs.twimg.com/media/HI0YtJKbMAIvt4p?format=jpg&name=large)

配置方式都一样，到对应平台注册拿 Key → 在 Copilot 加 Custom Model → 填三项 → 保存。

怎么用

- **聊天面板：** 右侧气泡图标 → 输入框打字。常用指令："帮我总结成 3 个要点：……"、"翻译成英文：……"、"基于这个大纲写一篇文章：……"。
- **选中处理：** 编辑区选中一段文字 → 右键 → Copilot 相关菜单。
- **命令面板：** Ctrl/Cmd + P → 搜 copilot 看所有命令。

排错清单

1. API Key 复制完整了吗？前后有没有空格？
2. Base URL 结尾的 /v1 是不是多加或少加 /？
3. Model Name 一字不差？建议直接复制表格里的。
4. 账户还有余额吗？
5. 海外模型必须开科学上网。

## 第 11 章 同步与备份

笔记是越攒越值钱的资产，**一定要做好备份**。

- **Obsidian Sync（官方付费）：** 端到端加密、版本历史、跨设备无缝。最省心，每年几百块。
- **坚果云 / iCloud：** 把仓库放进它们的同步目录。简单但 iCloud 仅限苹果。
- **Obsidian Git 插件 + 私有 GitHub 仓库：** 同步 + 完整版本历史，能回滚到几个月前。适合愿意稍微折腾的人。
- **Syncthing：** 完全免费、开源、点对点。配置稍麻烦但不依赖任何云厂商。

**底线：不管用哪种，至少保留一份独立的本地备份**（比如每周 rsync 到外置硬盘）。

## 第 12 章 新手学习节奏

**别贪快。Obsidian 的价值随时间累积。** 一周写 10 篇可能没感觉，坚持一个月、三个月，你会真的拥有"第二大脑"。

![图像](https://pbs.twimg.com/media/HI0Yr20bkAAxgwb?format=jpg&name=large)

5 条核心心法

1. **先写再整理。** 别先花两小时设计文件夹分类。写多了自然知道怎么整理。
2. **不要追求完美的笔记。** 一行字也值得记。笔记是给自己用的，不是作品。
3. **养成链接的习惯。** 想到"这跟之前什么有关"就顺手 \[\[\]\]。
4. **新手期别装太多插件。** 先裸用 1-2 周。
5. **不要把 Obsidian 当收藏夹。** 原封不动复制粘贴等于零，**用你自己的话重新表达**才有价值。

搭建完知识库后，你就可以按照我之前帖子里写的方法15分钟创作一篇长文了：

> 5月1日

## 第 13 章 常见问题 FAQ

**Q：数据安全吗？会不会丢？**

笔记是本地 .md 文件，不依赖任何云端，软件公司倒闭也照样能用文本编辑器打开。**但你要自己做好备份**——本地文件丢了就真丢了，参见第 11 章。

**Q：我不懂 Markdown 怎么办？**

实时预览模式下你几乎感觉不到自己在写 Markdown。三天就全记住了，没学习门槛。

**Q：旧笔记能从其他工具导入吗？**

Notion 官方有专门工具；印象笔记、Bear、Apple Notes 都有迁移方案或社区插件。**但建议先用 Obsidian 建立新习惯，再慢慢迁移旧笔记。**

**Q：社区插件安全吗？**

开源，官方有基本审核。绝大多数安全。建议只装下载量大、评价好、更新频繁的插件。

**Q：插件市场打不开？**

国内访问 GitHub 偶尔慢。切换网络或开加速工具；也可以去 PKMer 等镜像站下载文件，手动放到 仓库路径/.obsidian/plugins/插件名/ 后重启。

**Q：MOC、PARA、Zettelkasten 我该选哪个？**

不是非要选。先写半个月，看哪种需求自然冒出来再说。三种可以混着用。

我的仓库/
├── 00-收件箱/      # 临时素材，待整理
├── 01-日记/        # 自动按 YYYY-MM-DD 命名
├── 02-周记/        # YYYY-Www
├── 03-项目/        # PARA 的 Projects
├── 04-领域/        # PARA 的 Areas
├── 05-资源/        # PARA 的 Resources
│   ├── 读书笔记/
│   ├── 知识卡片/
│   └── 学习/
├── 06-归档/        # PARA 的 Archives
├── 99-附件/        # 图片、PDF 等
└── 模板/           # 各类笔记模板

数字前缀让排序固定。**文件夹不超过 1-2 级**，同主题用 MOC 串起来比建嵌套子文件夹好。实在拿不准的笔记先丢 00-收件箱，定期清理。

从今天开始就好。一篇日记、一个想法、一句话都行。
