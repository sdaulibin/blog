> 你们的关注和支持、转发、点赞、收藏和评论都将成为我创作最大的动力。

我第一个 100 万，不是靠流量密码赚来的。

更准确地说，它是我第一次意识到，**一个普通人想从 0 做出结果，真正要卖掉的不是时间，也不是服务，而是自己的判断力。**

100 万这个数字放在今天不算夸张。和那些年入千万、融资几轮、团队几十上百人的创业故事相比，它甚至很朴素。

但对我来说，它很重要。

因为那是我第一次完整经历一门生意从无到有的全过程：没有客户，没有体系，没有团队，到后来开始有稳定收入、有交付流程、有口碑、有转介绍，也慢慢有了团队协作。

这篇不会写成“手把手教你复制我的生意”。

有些具体打法、渠道、产品结构，我之后可能会放到商业咨询或者“从 0 到 A7”的系统课里讲，因为那些东西如果展开，会非常长，也需要结合具体业务来拆，单独写在文章里反而容易变成碎片化信息。

但我可以先把最核心的东西讲清楚。

**从 0 到 100 万，最关键的不是获客技巧、销售话术、定价套路，而是你能不能完成三次升级：从卖时间，到卖判断，再到卖系统。**

## 一、不要一开始就问怎么获客

![图像](https://pbs.twimg.com/media/HIGymGHacAAIpZ0?format=jpg&name=large)

很多人一开始创业，会特别执着于“怎么获客”。

这个问题当然重要，但它排得太靠后了。

真正排在前面的，是你有没有找到一个足够真实、足够具体、足够痛，并且对方愿意为解决它付费的问题。

我最早做留学相关服务的时候，没有一上来就想做一个机构，也没有什么宏大的商业计划书，更没有完整的产品矩阵。最开始其实很简单，就是身边有人真的需要帮助，而我刚好能提供比市场上更细、更真、更有判断力的解决方案。

对留学咨询来说，客户真正买的不是信息。

信息网上都有。

**客户真正买的是判断、路径、陪伴、风险控制和结果概率的提升。** 尤其是面对申请、选校、文书、研究计划、面试、教授沟通这些事情，很多学生和家长手里并不缺资料，真正卡住他们的是资料之间怎么连接，下一步该优先做什么，以及一个选择背后的风险到底有多大。

所以我最早做的事情，其实不是包装自己。

是不断回答真实问题。

有人问，我就认真讲；有人拿背景来咨询，我就帮他拆；有人不知道要不要走日本、香港、新加坡、欧美，我就根据他的条件、预算、时间、语言、专业和职业目标去分析。

这些东西看起来很笨。

但在从 0 到 1 的阶段，它很有效，因为你没有品牌，没有案例，没有团队，没有规模，唯一能拿出来的就是你这个人本身的判断力。

**早期客户买单，很多时候看重的不是你看起来多大，而是他能不能感受到，你真的理解他的问题，也真的有能力带他走下一步。**

## 二、获客的本质，是把信任放到成交前面

![图像](https://pbs.twimg.com/media/HIGyog8bcAAKHH3?format=jpg&name=large)

后来我慢慢意识到，获客不是把陌生人说服成客户。

更准确地说，**获客是把一部分本来就有需求、但还没有建立信任的人，通过内容、沟通、案例、口碑和服务触点，一步步筛选出来。**

我早期的获客方式并不复杂。

持续输出，认真沟通，做好交付。

持续输出，是为了让别人知道你是谁。如果你不表达，市场根本不知道你存在。很多人能力很强，但一直藏在后面，总觉得等自己再成熟一点、案例再多一点、体系再完整一点再出来，结果等来等去，别人已经通过持续表达抢占了心智。

这点其实很像《纳瓦尔宝典》里讲的“专长”和“杠杆”。真正能变成财富的，往往不是一个随处可替代的技能，而是你长期积累出来、别人很难复制的具体判断力；而内容，就是普通人最低成本的杠杆之一，它让你的判断不再只停留在一对一沟通里，而是可以被陌生人反复看见、反复验证、反复建立信任。

认真沟通，是为了让信任落地。

内容只能建立初步信任，真正的成交往往发生在具体沟通里。客户会看你怎么理解他的背景，怎么判断他的风险，怎么指出他的误区，怎么回应他的焦虑，也会看你到底是急着收钱，还是愿意在成交前就把关键问题讲清楚。

做好交付，是为了让获客变成复利。

早期最强的获客方式，其实是把前几个客户服务好。因为每一个满意客户，背后都可能带来转介绍、案例、口碑和信任背书；每一个糟糕交付，也会反过来侵蚀你的未来。

所以我很早就有一个朴素判断：**咨询生意的增长，不应该只靠前端销售，它必须靠后端交付产生复利。**

查理·芒格一直很强调复利，也很强调避免愚蠢。放到咨询生意里看，复利不是一句漂亮话，它其实很具体：**每一次交付都要让下一次交付更容易，每一个案例都要让下一次获客更可信，每一次踩坑都要让系统少犯一次同样的错。**

## 三、客户到底从哪里来

很多人问客户从哪里来，其实这个问题不能只看渠道。

因为同一个渠道，不同人的转化率可能完全不一样。你在小红书发内容能来客户，别人也在小红书发内容却没人问；你靠转介绍能成交，别人靠转介绍来的都是低质量线索。原因不只是平台不同，而是你有没有在渠道背后建立信任资产。

以我自己的经验，早期客户主要来自几类地方。

第一是内容。内容让陌生人第一次知道你，也让暂时不成交的人持续观察你。很多客户不会看一篇文章就付费，他可能看了你很多篇内容之后，慢慢确认你靠谱，最后在某个节点来找你。

第二是转介绍。一个客户如果真的被服务好，他会把你推荐给同学、朋友、学弟学妹、家长圈子。转介绍的质量通常比冷流量更高，因为它天然带着信任背书。

第三是圈层。当别人一想到某个具体问题，就想到你，这就是圈层心智。

第四是案例。案例不只是“某某拿到了 offer”这种结果展示，更重要的是把过程讲清楚：起点是什么，难点是什么，策略是什么，关键动作是什么，最后为什么能成。

第五是私域。私域的价值不在于朋友圈刷屏，而在于长期信任维护。很多人当下没有需求，但他会长期看你，等自己或者身边人有需求时再来找你。

所以客户从哪里来？

**表面上来自内容、转介绍、圈层、案例和私域。本质上来自你长期积累的信任资产。**

如果没有信任，渠道只是流量。

**有了信任，流量才会变成客户。**

## 四、定价不是胆子大，而是机制设计

![图像](https://pbs.twimg.com/media/HIGysmxboAAf921?format=jpg&name=large)

我早期也踩过定价的坑。

一开始很容易不好意思收费，总觉得自己还不够大、案例还不够多、品牌还不够响，所以价格要低一点，服务要多一点，承诺要满一点。

这几乎是所有知识服务和咨询创业者都会经历的阶段。

但后来我慢慢明白，低价并不一定带来轻松，很多时候反而会带来更重的消耗。

**价格不只是收入数字，它也是筛选机制。**

一个服务如果定价太低，你会吸引来大量不尊重专业、不理解服务边界、只想用最低成本获得最高确定性的人。你以为自己在降低门槛，实际上可能是在提高交付风险。

所以我后来思考定价，不再只看“别人卖多少钱”，也不只看“客户能不能接受”，而是看几个维度：问题复杂度、交付强度、风险责任、客户适配度。

越复杂的问题，越不能用简单信息服务定价。比如一个学生背景复杂、目标跨度大、时间又紧，需要做多国混申、研究方向定位、文书策略、面试训练、教授沟通和风险预案，这种服务本身就不是“帮你填几个表”。

芒格讲商业时很重视“激励机制”，因为激励会决定人怎么行动。定价其实也是一种激励机制：价格太低，会吸引一批只想低成本占用你时间的人；价格太高但交付跟不上，又会逼着你用包装去掩盖能力不足。

**好的定价不是把钱收高，而是让客户、团队、交付强度和长期口碑之间形成正确激励。**

真正健康的定价，应该让客户觉得值，也让服务方有足够空间把事情做好。

## 五、从卖自己，到卖体系

![图像](https://pbs.twimg.com/media/HIGyu_ZaYAA3-4X?format=jpg&name=large)

从 0 到 100 万的过程中，我最大的一个转变，是从“靠自己扛”慢慢转向“靠体系支撑”。

最开始当然是卖自己。

客户信任的是你本人，沟通找你，方案找你，关键判断找你，出了问题也找你。这在早期很正常，因为你没有团队，没有品牌，没有流程，客户买的就是你这个人的经验和责任心。

但这个阶段有一个天然天花板。

**你的时间有限，注意力有限，精力有限，情绪也有限。** 如果所有事情都压在你身上，业务越增长，你越痛苦。表面上收入提高了，实际上人被系统拖着走。

所以后来我开始做几件事。

沉淀 SOP。哪些问题每个客户都会遇到，哪些材料每次都要解释，哪些节点每次都要提醒，哪些风险每次都要提前说清楚，这些东西不能每次靠脑子重新讲，必须写下来、流程化、模板化。

沉淀案例库。案例库的价值不在炫耀，而在提高判断效率。你见过足够多类似背景、类似目标、类似问题之后，就能更快判断一个学生适合什么路径，哪里有机会，哪里有风险。

沉淀知识库。项目信息、申请要求、教授方向、文书结构、面试问题、时间线、材料清单，这些如果不沉淀，就会变成个人记忆负担；一旦沉淀下来，就会变成团队共享资产。

沉淀产品结构。不能所有客户都靠临时沟通来判断服务内容。你需要有清晰的产品层级，什么适合轻咨询，什么适合完整规划，什么适合高强度陪跑，什么应该拒绝。

沉淀工具。我现在对 AI 和自动化很感兴趣，很大一部分原因也来自这里。咨询业务里有大量重复劳动，如果这些重复劳动不能被工具、模板、系统消化，创始人和团队就会被琐事拖死。

所以从 0 到 100 万，不只是收入增长。

**更重要的是你要完成一次身份变化：从一个能解决问题的人，变成一个能把解决问题的方法沉淀成系统的人。**

前者可以养活自己。

后者才有可能养活团队。

## 六、独立开发的意义，是业务控制权

我后来越来越重视自己做系统、做工具、做自动化，倒不是因为我想成为程序员，也没有觉得所有东西都要自己开发。

更根本的原因是，独立开发让你拥有更强的业务掌控感。

很多小团队最大的问题，是创始人的想法停留在脑子里，团队靠口头传递，流程靠人盯人，数据靠表格临时拼，客户状态靠记忆和聊天记录回想。

这种方式在早期能跑。

但一旦客户数量上来，问题就会集中爆发：谁跟进到哪一步，哪个材料还没交，哪个节点快截止，哪个学生风险升高，哪个顾问需要支持，哪个流程反复卡住，如果没有系统承接，最后都会变成创始人的精神负债。

**一个好的内部系统，至少要让状态可见、流程可复用、知识可调用、判断可辅助。**

《纳瓦尔宝典》里讲过很重要的一点：代码和媒体，都是不需要别人许可就能使用的杠杆。以前一个小团队想放大自己，往往只能靠招人、租办公室、扩销售，但现在不一样，**如果你能用 AI、代码、内容和自动化，把自己的判断封装进系统里，你就能获得一种非常适合一人公司和小团队的杠杆。**

这就是独立开发对我的意义。

**它不是技术崇拜。它是业务控制权。**

## 七、团队化最难的，是放弃“只有我最懂”

从一个人做到团队，最大的难点不是招人。

是创始人要放弃一种幻觉：只有我最懂，只有我做才放心，只有我亲自盯着才不会出错。

这个想法在早期当然有一定道理。业务确实是你跑出来的，客户确实是信任你，很多判断也确实来自你的经验。但如果你一直停留在这里，团队永远长不出来。

团队化不是简单地把工作分出去。

**团队化的本质，是把你的判断标准、工作流程、质量要求和价值观，变成别人也能理解、执行和迭代的东西。**

这就要求你明确标准，拆分流程，允许试错，建立复盘机制。

什么叫好方案，什么叫好文书，什么叫有效沟通，什么叫风险提示到位，不能只靠一句“你再改改”。标准越模糊，团队越依赖你。

这也是芒格式的反向思考：如果我想让一个团队永远长不出来，那我只需要让所有判断都回到我这里，让所有标准都只存在我脑子里，让所有问题都靠临场救火。

反过来看，想让团队真正长出来，就必须把判断外显，把标准写下来，把错误变成复盘，把经验沉淀进系统。

## 八、给想从 0 到 A7 的人的一点建议

如果你现在也想做一个自己的小生意，尤其是咨询、知识服务、个人 IP、一人公司这类业务，我会建议你先别急着问怎么涨粉、怎么成交、怎么定价。

先问自己几个更底层的问题。

**我解决的问题，别人是否真的愿意付费？我是否能在这个问题上提供比普通信息更有价值的判断？我有没有一个可以持续建立信任的内容入口？我的服务能不能沉淀成流程、案例、知识库和产品结构？如果客户数量变多，我能不能不靠自己硬扛，而是通过团队和系统交付？**

这些问题如果想不清楚，流量越大，反而可能越危险。

因为流量会放大一切。

**它会放大你的价值，也会放大你的混乱；会放大你的专业，也会放大你的交付漏洞；会放大你的收入，也会放大你的情绪消耗和管理问题。**

所以从 0 到 100 万，真正要练的不是某个技巧。

**是把自己变成一个能被市场信任的人，再把这个人的能力，逐步沉淀成一个可复制、可交付、可管理、可持续的系统。**

## 结尾

我现在越来越觉得，创业早期最重要的能力，是诚实地面对市场。

市场不会因为你努力就给你钱。

市场也不会因为你觉得自己值得，就自动认可你的定价。

**市场只会持续问你几个很现实的问题：你能解决什么问题？你解决得是否比别人好？别人为什么信你？你的服务是否稳定？你能不能在更多客户身上重复交付这种价值？**

从 0 到 100 万，就是不断回答这些问题的过程。

是用内容回答，用沟通回答，用成交回答，用交付回答，用案例回答，用团队回答，用系统回答。

如果未来我要做商业咨询，或者做“从 0 到 A7”的训练，我真正想教的也不是所谓暴富技巧。

我更想讲的是，**一个普通人如何从自己的经验出发，找到真实需求，建立信任，设计产品，完成定价，搭建系统，使用 AI 和工具放大自己，最后从单点能力走向稳定的商业结构。**

这件事不性感。

但它真实。

而真实的东西，才有复利。

> **从 0 到 100 万，关键不在于把自己包装成一个厉害的人，而是先真的解决一个问题，再把解决问题的能力沉淀成系统。**

**早期你卖的是自己，中期你卖的是判断，后期你卖的是体系。能不能从 0 走到 A7，关键就在于你是否完成了这三次升级。**
