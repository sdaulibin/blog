我现在两个 Claude Code Max Plan 同时跑，加上 Codex 处理并行任务，过去 4 个月用这套组合上了二三十个 SEO 站、搭了一套内容生产系统、做了大量的项目。最近 Codex 能力提升很快，很多任务我已经放到 Codex 上跑了，但 Claude Code 依然是我的主力工具——它是整个 AI 编程工作流的起点。

但最开始我也是什么都不懂，从终端里敲下第一行 claude 开始摸索的。踩了不少坑，也花了不少冤枉钱。这篇把我希望自己当初就知道的东西全写出来，从安装到真正用起来，能帮你少走几个月弯路。

## Claude Code 到底是什么

如果你一直在浏览器里用 Claude 聊天，那你只用到了它大概 10% 的能力。。。

浏览器里的 Claude 是你问它问题，它给你回答。Claude Code 完全不一样——它直接在你的电脑上干活。读你的文件、改你的代码、跑你的脚本、帮你部署，你的整个项目它都能看到，不用你一段一段粘贴代码过去了。

这两个东西的差距，说实话我刚切过来的时候自己都吓了一跳。

现在 Claude Code 有三种形态：

![图像](https://pbs.twimg.com/media/HHCR6CfaQAEgeZL?format=jpg&name=large)

**CLI（命令行）**——在终端里输入 claude 就能用。这是最原始也是功能最全的方式。我日常主要用这个。

**Desktop App**——4 月 15 号刚改版的桌面应用，有图形界面，支持并行开多个会话。不想碰终端的人可以从这个开始。

**VS Code 扩展**——装在编辑器里，写代码的时候直接在侧边栏用。

三种形态底层能力是一样的，不存在谁比谁强。如果你是程序员或者不怕终端，直接上 CLI，体验最好。如果你看到黑色的终端窗口就头大，Desktop App 完全没问题，4 月刚改版的界面做得很不错。我个人一直用 CLI，因为可以同时开好几个终端窗口跑不同的任务，效率确实高。

**怎么装**

安装这一步我不打算写详细教程，因为 2026 年了，安装软件这件事直接问 AI 就行了。

你把"我用的是 Mac / Windows，帮我安装 Claude Code CLI"这句话发给任何一个 AI（ChatGPT、豆包、Kimi 都行），它会一步一步教你。中间碰到任何报错，截图发给 AI，它也能帮你解决。

这个过程本身就是在练习和 AI 协作，比看我写的安装教程有用得多。

三种形态都可以这么装：CLI 是命令行安装，Desktop App 去官网下载，VS Code 扩展在插件市场搜索安装。具体步骤交给 AI。

**订阅方案**

这个部分我觉得很多教程都不太写，但其实新手最懵的就是这个——到底要花多少钱？

![图像](https://pbs.twimg.com/media/HHCR9LOaQAAyF79?format=jpg&name=large)

**Max Plan（$100/月或$200/月）**——官方订阅，最简单，登录就能用。$100 的额度对轻度使用够了，$200 的我一个人能跑满。我现在两个 $200 的 Plan 同时用，月支出 $400。

**API 中转站**——不订阅官方 Plan，直接用第三方 API。成本低很多，而且有效避免风控，但需要自己配置。适合预算有限的朋友或者实在不想折腾风控的朋友。一些中转站的 Claude 模型价格大概是官方的 1/3 到 1/5。并且问题是良莠不齐，必须得自己真金白银试才能试出不跑路的中转站。我从去年年中开始使用中转站，光见证的跑路掉的不下10家，自己被割韭菜的也有两三家，几千块钱，朋友们真的需要谨慎。我的感受是，找一个好的中转站，和相亲一样 o(╥﹏╥)o

**新手建议**：如果你就是想试试水，先用官方的 $100 Max Plan。什么都不用折腾，装完就能用。等用顺了再考虑要不要切中转站。

## 装完第一件事

我见过太多人装完 Claude Code 第一件事就是上来说"帮我写一个网站"、"帮我做一个 xxx"。

不是不行，但你会浪费掉 Claude Code 最强的一个能力。

装完之后你应该做的第一件事是搞一个 **CLAUDE.md** 文件。

![图像](https://pbs.twimg.com/media/HHCSAHqbgAA15Lc?format=jpg&name=large)

CLAUDE.md 是 Claude Code 和普通 AI 聊天最根本的区别。它是一个放在你项目根目录的文件，Claude Code 每次启动都会读它。相当于你给 AI 定了一份规矩，它每天上班第一件事就是读这份规矩。

没有 CLAUDE.md 的 Claude Code，就是一个什么都不知道的新员工，每次你都得从头交代。有了就不一样了。

**第一步：直接用别人写好的**

不用自己从零写。GitHub 上有很多高质量的 CLAUDE.md 模板，搜 "CLAUDE.md template" 或者 "everything claude code" 就能找到一堆。

比如有个叫 Everything Claude Code 的开源项目（Anthropic 黑客松获奖作品），里面包含了完整的 CLAUDE.md 模板、Agent 配置、Skill 模块、Hook 自动化，全套拿来就能用。

新手直接 clone 一个这种项目，把它的 CLAUDE.md 复制到你自己的项目根目录，就能立刻感受到 Claude Code 的不同。

**第二步：改成适合自己的**

用了几天之后你会发现，别人的 CLAUDE.md 有些规则不适合你的项目，有些你需要的规则它没有。这时候开始改。

怎么改？最简单的办法：直接跟 Claude Code 说。

比如你发现它每次提交代码都用英文写 commit message，但你想用中文——直接告诉它"在 CLAUDE.md 里加一条：提交信息用中文"。它会自己去改 CLAUDE.md。

再比如你觉得它改代码太激进了，老是大范围重构，你可以说"在 CLAUDE.md 里加一条：改动范围要小，一次只改一个功能，改之前先让我确认"。

CLAUDE.md 是活的。用的过程中觉得哪里不对就加规则，Claude Code 犯了错就补一条。我自己的 CLAUDE.md 经过几个月迭代已经变成了一个不短的文件，包含了项目结构、工作流程、代码规范、部署方式、凭证管理。

核心思路是：**你不需要一次写到位，你只需要每次它做错的时候加一条规则。** 用的越久，它就越懂你。当然，规则太多了之后还是需要整理一下的，不过那时你一定有丰富的经验来处理他了。

**真正开始干活**

CLAUDE.md 写好之后，cd 到你的项目目录，输入 claude，就可以开始了。

Claude Code 和普通聊天 AI 最大的区别是它能执行操作。你可以说：

- "帮我看下这个项目的目录结构"
- "src/index.ts 第 30 行有个 bug，帮我修一下"
- "帮我写一个部署脚本，部署到 Cloudflare Pages"
- "帮我把所有的 console.log 都删掉"

它会直接去读文件、改文件、跑命令。每次操作前会先告诉你它要做什么，你按回车确认就行。

还有一个我花了不少时间才真正学会的事情：**不要一次给它太大的任务。** 说"帮我做一个完整的电商网站"，出来的东西大概率不能用。但说"帮我先把登录页面做出来，用 Next.js + Tailwind"，它能做得很好。

拆任务这个事情说起来简单，真正做到位需要练。但一旦你拆任务的感觉上来了，Claude Code 的产出质量会有一个质的飞跃，到时候你就会明白我在说什么了。

## Skill：Claude Code 真正强大的地方

很多人用了一段时间 Claude Code 之后会问：它和在浏览器里聊天比，真的已经很强大了，但是我们可不可以更强呢？

有的！朋友！有的！那就是Skill！

**Skill 是什么**

![图像](https://pbs.twimg.com/media/HHCSR4BbIAA7eL6?format=jpg&name=large)

简单说，Skill 就是你教给 Claude Code 的一套做事方法。你把某个任务的完整流程写成一个 Markdown 文件，Claude Code 以后碰到类似的事情就按这个方法来。

举个例子。我有一个部署 Skill，里面写了"读取凭证 → 构建项目 → 部署到 Cloudflare → 验证部署结果"这一整套流程。以前每次部署我都得跟 Claude Code 解释一遍怎么部署、凭证在哪、用什么命令。有了这个 Skill 之后，我说一句"部署"，它就知道该怎么做了。

再比如我有一个 SEO 审计 Skill。一行命令它就能把一个网站的 SEO 问题全部扫一遍——技术 SEO、Schema 标记、页面质量、外链状况，最后输出一份报告。这个流程如果我自己手动做，大概需要两三个小时。

**不需要自己写，直接用别人的**

这是最重要的一点：**GitHub 上有大量现成的 Skill，直接拿来用就行。**

搜 "claude code skills" 能找到各种各样的——SEO 审计、代码审查、内容创作、数据分析、自动化部署。都是别人实战总结出来的，经过验证的。

安装也不复杂。大部分 Skill 就是一个 Markdown 文件，你把它放到 ~/.claude/skills/ 目录下就行了。有些更完整的 Skill 会附带脚本和配置文件，clone 下来按 README 操作。

我自己目前常用的 Skill 大概有三四十个，覆盖了部署、SEO、内容创作、图片生成、代码审查、数据库操作这些场景。一半是从 GitHub 上找的，另一半是自己根据需要写的，我觉得算不算熟练使用 Claude code 的一个重要区分点就是是不是开始自己产出 skill 自己用了，不过自己产出好的 skill 属于比较资深的内容，我觉得在进阶的去写比较好。

**怎么找到好的 Skill**

几个我觉得靠谱的来源：

1. GitHub 搜 "claude code skills" 或 "claude code awesome"，按 star 数排序
2. X 上搜 "claude code skill"，看谁在分享自己用的 Skill
3. Everything Claude Code 这个项目里有一整套经过验证的 Skill 集合
4. dontbesilent 的 dbs skill 系列，我写文章的时候用了大量他的 Skill，效果非常好

**重要提醒：装 Skill 之前做安全审计**

这一点必须说。Skill 本质上就是一段指令，Claude Code 会照着执行。有不法分子会在 Skill 里埋后门——比如让 Claude Code 偷偷把你的 API Key 发到外部服务器，或者在你不知道的情况下执行危险命令。

我自己的做法是：任何从外面拿来的 Skill，装之前先让 Claude Code 自己做一遍安全审计。直接跟它说"帮我审计一下这个 Skill 文件，看有没有安全风险，有没有往外发数据的行为，有没有执行危险命令"。Claude Code 读代码的能力很强，大部分明显的后门它能扫出来。

另外几个判断标准：star 数太少的项目谨慎用，没有 README 的项目不要用，Skill 文件里出现了你看不懂的外部 URL 要格外小心。这个圈子整体还是好的，但防人之心不可无。。。

**用了 Skill 之后的变化**

没装 Skill 的 Claude Code，就像一台刚出厂的手机——能打电话能发短信，但你不会觉得它有什么了不起。装了 Skill 的 Claude Code，就像手机装满了 App——它突然变成了一个能干任何事情的工具。

我的感受是，Claude Code 裸跑和装了 Skill 跑，完全是两个东西。如果你用了 Claude Code 觉得"也就那样"，大概率是因为没装 Skill。

**Custom Command 顺便提一下**

和 Skill 相关的还有一个叫 Custom Command（自定义命令）的东西，比 Skill 更轻量。适合把一个简单的重复操作变成一个斜杠命令。

比如我做内容的时候给自己做了 /记录选题、/深化选题、/生成标题 这些命令，输入一个斜杠就触发。做法也简单：在 .claude/commands/ 目录下放一个 Markdown 文件，文件名就是命令名。

怎么理解 Skill 和 Command 的区别呢。。。Skill 是一套完整的能力，Command 是一个快捷操作。新手先把别人的 Skill 装上用起来，等你发现"这个操作我天天在做"的时候，再去搓一个 Command。不着急，用到了自然就会了。

## 新手容易踩的坑

![图像](https://pbs.twimg.com/media/HHCSfxFbUAAIXUf?format=jpg&name=large)

**不写 CLAUDE.md 就开始干活**

我见过最多的情况就是这个。装完 Claude Code 直接上手干活，干了一周发现它的输出质量飘忽不定——有时候特别好，有时候完全跑偏。然后就觉得"Claude Code 也就那样"。

其实不是它不行，是它不知道你想要什么。你没给它规矩，它只能猜。写了 CLAUDE.md 之后，输出稳定性会有一个肉眼可见的提升。

**一次给太大的任务**

我刚开始用的时候也犯过这个错。让它"帮我做一个完整的数据看板"，结果出来的东西前后端风格不统一、数据库字段乱取名、接口设计一塌糊涂。。。后来我学会了拆——先做数据库设计让我看，没问题再做后端接口，接口跑通了再做前端。同样的项目，拆成小步骤之后交付质量完全不一样。

**备份，备份，备份**

Claude Code 降智的时候真的会干掉你的数据。不是开玩笑，我经历过。

代码方面，一定要用 git，而且 git 仓库一定要推到云端（GitHub、GitLab 都行）。只在本地 git commit 不 push 的话，硬盘一坏全没了。Claude Code 改代码改出问题很常见——改着改着把配置文件搞坏了，删掉了不该删的东西。有 git 就能回滚，没 git 就只能重写。

数据库比较特殊，数据库没法 git。如果你的项目有数据库，必须单独做定时备份。我的做法是每四个小时备份一次关键数据文件，同步到 Google Drive。Claude Code 有时候会写出覆盖数据的脚本——我有一次让它写了个新的数据处理脚本，直接覆盖了正在运行的旧脚本的输出文件，数据没了。。。从那以后所有数据文件操作前先 .bak 备份。

总结就是：代码上 git + push 云端，数据库定时备份，数据文件操作前先 bak。这三条全是惨痛历史换来的。

**账号风控**

如果你的 IP 不干净（用的是机场的共享 IP），Anthropic 可能会封号。我自己 Claude 账号倒是没出过问题（人在新加坡 IP 干净），但圈子里被封的不少。有人一个月封了近 20 个号。。。

建议用住宅 IP 或者稳定的独立 VPS。如果在国内，这个成本要算进去。科学上网 + 纯净 IP 是用 Claude Code 的前置条件，不是可选项。

## 我的真实感受

从 2023 年底的 ChatGPT 到 2024 年的 Cursor 再到 2025 年 4 月开始用 Claude Code，三次切换。最后留在 Claude Code 的原因很简单：它真的能帮你干完一件事，不是给你一段代码让你自己粘贴。当然目前 codex 几乎杀疯了，而且 codex 的风控明显更友好，实在被 claude code 这个小妖精折磨的不行的朋友不妨试一下 codex，近期我一定也会把 codex 的实践教程更出来的。

对我来说 Claude Code 不是一个"帮我补全一行代码"的工具，它更像一个工程协作伙伴。我给它一个任务，它能通读整个项目，理解几十个文件之间的依赖关系，然后完成一组完整的操作。这种体验一旦习惯了就回不去了。。。

**对新手来说，最重要的就两件事：**

1. 搞一个好的 CLAUDE.md
2. 学会使用skill

这两件事做到了，Claude Code 就已经能帮你做很多事情了。剩下的那些进阶玩法，用着用着自然会摸出来的，不用着急。
